Carl --

 

Please find below the project outline that we discussed yesterday (Please forward this on to Lucaz too please, as I can’t find his email address):

 

1...Set-up an IIS Machine:

This should be at least Windows 2003 server, with a reasonable amount of RAM and sisk space availible. Ideally this machine should have no other function on the network, but the server can run on the Priority Machine if required.

 

2...Set up the A-records to point to the IIS Machine:

We need an A-record both on the local domain and with your Internet DNS provider that will be used to connect to the mobile service, eg mobile.dunlop.com.

 

Local DNS should resolve this to the network address of the IIS server. Internet DNS should resolve this name to the external IP address of your firewall. You firewall should be configured to pass traffic on port 8080 to the internal network address of the IIS server.

 

After testing remote access to the mobile service can be removed if required.

 

3...Configure remote Access to allow configuration:

We will need to connect to the IIS server to install all the components for the mobile service. Please ensure that we can access the IIS machine from outside the network.

 

4...Install Wireless Access in Goods Inwards:

Prior to go live we will need wireless coverage of the area in which the devices are to be used. I'm sure Lacasz can get this organised.

 

5...Remote configuration the IIS Machine:

Once the IIS machine is set up I will dial in and install the required services. In the first instance, and for testing

 

6...Customisation of the PO Label site:

We will require from yourselves a label design, detailing what elements should be present on the purchase order labels that can be downloaded from the service. Once received we will use these specifications

 

7...Create an Emulation environment (for testing on the PC):

For testing I will be providing win 32 builds of the application, to enable you to test the application prior to receipt of the device. The builds will be specific to Dunlop, and will need to be tested against your service, so this phase will start once your server is installed.

 

8...Order the device:

I have spoken to Luscz with regards this, and we are going to source the device. Lucasz is deciding which features he requires and is coming back to me with the relevant paperwork.

 

9...Install the emulator build on the device:

Once we are happy with the emulation environment we will load a build image onto the actual device.

 

10...Go Live:

While the actual go-live date is dependant upon the steps above, when we get past step 5 we should have enough information to schedule the go-live. I will be on-site for this part of the process.

 

Any questions or comments please don't hesitate to come back to me on the usual number.
